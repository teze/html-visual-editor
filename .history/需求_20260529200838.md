# HTML 可视化编辑器 · 产品需求文档

## 一句话定位

> 打开任意 HTML 文件，在渲染后的页面上直接点击编辑，无需接触代码，编辑完导出干净的 HTML 文件。

---

## 背景 & 痛点

AI 时代大量工具（ChatGPT、Claude 等）可以直接生成 HTML 页面，HTML 正在成为一种通用的内容交付格式。但生成之后的**微调编辑**存在门槛：

- 改文字需要找到对应代码行
- 调颜色、字号需要懂 CSS
- 非技术用户完全无法独立操作

**目标用户**：用 AI 生成了 HTML 页面，想自己改改内容，但不会写代码的人。

---

## 核心用户流程

```
1. 打开工具
2. 拖入 / 选择本地 HTML 文件
3. 看到和浏览器一样的页面渲染结果
4. 鼠标悬停 → 高亮元素边框
5. 单击元素 → 右侧面板显示该元素的可编辑属性
6. 在面板中修改 / 双击文字直接在预览中编辑
7. 修改实时反映在预览中
8. 点击「导出」→ 下载干净的 HTML 文件（不含编辑器注入代码）
```

## 技术要求

### 页面渲染

- 使用 `<iframe srcdoc="...">` 渲染用户的 HTML
- iframe 设置 `sandbox="allow-scripts allow-same-origin"` 保证 JS 可执行
- 注入编辑器脚本通过 `postMessage` 与父页面通信，避免跨域问题

### 编辑器注入脚本（iframe 内部）

- 监听 `mouseover` / `mouseout` → 悬停高亮
- 监听 `click` → 选中元素，通过 `postMessage` 将元素信息发给父页面
- 监听 `dblclick` → 激活 `contentEditable`
- 监听 `input` → 实时同步文字变更
- 监听 `message` → 接收父页面发来的属性更新指令，直接修改 `element.style[prop]`

### 关键技术注意点

- **脚本注入方式**：将编辑器函数用 `.toString()` 提取源码再拼接注入，避免字符串多层转义导致语法错误
- **颜色读取**：`getComputedStyle` 返回 `rgb()` 格式，需转换为 hex 以回填到颜色选择器
- **导出清理**：导出前移除注入的 `<script id="__htmledit__">` 标签，清除所有 `outline` / `cursor` 编辑态样式

### 技术栈建议

| 方案 | 适用场景 |
|------|----------|
| 纯 HTML + Vanilla JS | 单文件交付，最简单，0 依赖 |
| React + Vite | 需要组件化、状态管理时 |
| Electron / Tauri | 做成桌面应用，可访问本地文件系统 |
| VS Code 插件 | 在编辑器内嵌入预览 + 编辑面板 |

## 交互细节

- 悬停颜色：`rgba(0, 212, 170, 0.35)` 半透明
- 选中颜色：`#00d4aa` 实线 2px
- 修改立即生效，无需点击确认按钮
- 颜色选择器与 hex 输入框双向联动
- 不可编辑元素（`<html>` `<head>` `<style>` 等）跳过，不响应点击
- 导出后自动重新加载编辑器，保持可继续编辑状态
