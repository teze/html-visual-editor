# 撤销/重做和自动保存功能实现总结

## 完成时间
2026-05-31

## 实现的功能

### 1. 撤销/重做（Undo/Redo）

#### 核心功能
- ✅ 支持撤销操作（Undo）
- ✅ 支持重做操作（Redo）
- ✅ 最多保存 50 步历史记录
- ✅ 历史记录包含完整的 HTML 和选中状态
- ✅ 自动管理历史栈（超过限制时删除最旧记录）

#### UI 交互
- ✅ 顶栏添加"↶ 撤销"和"↷ 重做"按钮
- ✅ 按钮在无可用历史时自动禁用（半透明样式）
- ✅ 键盘快捷键：`Ctrl+Z` 撤销，`Ctrl+Y` 重做
- ✅ Mac 支持：`Cmd+Z` 和 `Cmd+Y`
- ✅ 在输入框中输入时禁用快捷键（避免冲突）

#### 集成点
- ✅ 每次属性修改后保存历史快照（`onChanged`）
- ✅ 删除元素后保存历史快照（`onDeleted`）
- ✅ 加载文件时清空历史并保存初始快照
- ✅ 撤销/重做时完整恢复页面状态

### 2. 自动保存（Auto Save）

#### 核心功能
- ✅ 修改后 2 秒自动保存到 localStorage
- ✅ 防抖保存机制（避免频繁写入）
- ✅ 保存完整的 HTML 内容
- ✅ 保存元数据（文件名、时间、大小）
- ✅ 刷新页面时自动检测并提示恢复
- ✅ 导出文件后自动清除保存记录

#### UI 交互
- ✅ 顶栏显示保存状态
- ✅ 状态文字：`未保存` / `保存中...` / `已保存 刚刚` / `已保存 X 分钟前`
- ✅ 颜色区分：灰色（未保存）、橙色（保存中）、绿色（已保存）
- ✅ 时间格式化：刚刚 / X 分钟前 / X 小时前 / 具体日期

#### 集成点
- ✅ 每次修改后触发自动保存（`onChanged`、`onDeleted`、`onTextInput`）
- ✅ 页面加载时检查是否有未保存内容
- ✅ 提示用户恢复未保存内容

## 技术实现细节

### 历史管理器设计
```javascript
// 历史栈结构
history = [
  { html: "...", selectedInfo: {...}, timestamp: 123456 },
  { html: "...", selectedInfo: {...}, timestamp: 123457 },
  // ...
]
currentIndex = 1  // 指向当前位置
```

- 使用数组作为栈
- `currentIndex` 指向当前位置
- `undo()` 时 `currentIndex--`
- `redo()` 时 `currentIndex++`
- `push()` 时删除 `currentIndex` 之后的所有记录

### 自动保存器设计
```javascript
// localStorage 存储
localStorage['html-editor-autosave'] = "<html>...</html>"
localStorage['html-editor-autosave-meta'] = JSON.stringify({
  filename: "test.html",
  savedAt: 1234567890,
  size: 12345
})
```

- 使用 `setTimeout` 实现防抖
- 每次修改时清除旧定时器，设置新定时器
- 2 秒后无新修改才真正保存

### 集成策略

**历史快照时机：**
- ✅ 属性修改成功（`onChanged`）
- ✅ 删除元素（`onDeleted`）
- ❌ 文字输入（`onTextInput`）- 不保存快照，避免过多记录

**自动保存时机：**
- ✅ 属性修改成功（`onChanged`）
- ✅ 删除元素（`onDeleted`）
- ✅ 文字输入（`onTextInput`）

## 已知限制

1. **历史记录**
   - 仅在当前会话有效，刷新页面后清空
   - 不支持跨文件历史记录
   - 最多 50 步，超过后删除最旧记录

2. **自动保存**
   - 仅保存最新版本，不支持多版本历史
   - localStorage 有大小限制（5-10MB）
   - 超大文件可能保存失败
   - 清除浏览器数据会删除保存内容

3. **性能**
   - 每次保存快照需要序列化整个 DOM
   - 大文件可能影响性能
   - localStorage 写入是同步操作

## 未来改进方向

1. **历史记录持久化**
   - 将历史记录也保存到 localStorage
   - 支持跨会话恢复历史记录

2. **增量快照**
   - 只保存变更的部分，而不是整个 DOM
   - 减少内存占用和序列化开销

3. **多版本管理**
   - 支持保存多个版本
   - 提供版本列表和对比功能

4. **云端同步**
   - 支持保存到云端
   - 多设备同步

5. **性能优化**
   - 使用 Web Worker 进行序列化
   - 使用 IndexedDB 替代 localStorage（支持更大容量）
