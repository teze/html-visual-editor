# 撤销/重做和自动保存功能说明

## 快速体验

1. 打开 `html-editor.html`
2. 加载 `test-undo-redo.html` 测试文件
3. 尝试以下操作：

## 撤销/重做功能

### 使用方法

**方式 1：点击按钮**
- 顶栏左侧有"↶ 撤销"和"↷ 重做"按钮
- 按钮在无可用历史时会自动禁用（半透明）

**方式 2：键盘快捷键**
- `Ctrl+Z`（Mac: `Cmd+Z`）：撤销
- `Ctrl+Y`（Mac: `Cmd+Y`）：重做

### 工作原理

- 每次修改（文字、样式、删除元素）后自动保存历史快照
- 最多保存 50 步历史记录
- 历史记录包含完整的页面状态和选中元素信息
- 撤销/重做时完整恢复页面到之前的状态

### 测试步骤

1. 修改标题文字 → 历史记录 +1
2. 修改段落颜色 → 历史记录 +1
3. 点击"撤销"或按 `Ctrl+Z` → 恢复到修改颜色之前
4. 点击"重做"或按 `Ctrl+Y` → 重新应用颜色修改
5. 观察按钮状态变化

## 自动保存功能

### 使用方法

**自动触发**
- 修改任何内容后，2 秒内无新修改时自动保存
- 顶栏右侧显示保存状态

**状态显示**
- `未保存` - 灰色，尚未加载文件
- `保存中...` - 橙色，正在保存
- `已保存 刚刚` - 绿色，刚刚保存成功
- `已保存 X 分钟前` - 绿色，显示上次保存时间

### 工作原理

- 使用浏览器 localStorage 存储
- 保存完整的 HTML 内容和元数据（文件名、时间、大小）
- 刷新页面时自动检测并提示恢复
- 导出文件后自动清除保存记录

### 测试步骤

1. 加载测试文件
2. 修改任何内容
3. 观察顶栏保存状态变化：`保存中...` → `已保存 刚刚`
4. 刷新页面（`F5` 或 `Cmd+R`）
5. 看到恢复提示，点击"确定"恢复内容
6. 验证修改是否保留

## 注意事项

### 撤销/重做
- ⚠️ 历史记录仅在当前会话有效，刷新页面后清空
- ⚠️ 在输入框中输入时，键盘快捷键不会触发（避免冲突）
- ⚠️ 加载新文件时会清空历史记录

### 自动保存
- ⚠️ 仅保存最新版本，不支持多版本历史
- ⚠️ localStorage 有大小限制（通常 5-10MB）
- ⚠️ 超大文件可能保存失败
- ⚠️ 清除浏览器数据会删除保存的内容

## 常见问题

**Q: 为什么刷新页面后历史记录消失了？**
A: 撤销/重做的历史记录存储在内存中，刷新页面会清空。但自动保存的内容会保留，可以恢复。

**Q: 自动保存的内容存在哪里？**
A: 存储在浏览器的 localStorage 中，与域名绑定。不同浏览器的数据是独立的。

**Q: 如何清除自动保存的内容？**
A: 导出文件后会自动清除，或者在浏览器设置中清除网站数据。

**Q: 撤销/重做支持跨文件吗？**
A: 不支持。每次加载新文件时会清空历史记录。

**Q: 键盘快捷键不工作？**
A: 确保焦点不在输入框中。如果在输入框中，快捷键会被禁用以避免冲突。

## 技术细节

### 历史管理器（history-manager.js）
- 使用栈结构存储历史快照
- 每个快照包含：HTML 内容、选中元素信息、时间戳
- 支持 undo/redo/canUndo/canRedo/clear 等操作
- 状态变化时通知 UI 更新按钮状态

### 自动保存器（auto-saver.js）
- 防抖保存：2 秒延迟，避免频繁写入
- 存储键：`html-editor-autosave`（内容）、`html-editor-autosave-meta`（元数据）
- 元数据包括：文件名、保存时间、文件大小
- 支持 scheduleSave/saveNow/restore/clear 等操作

### 集成方式
- 在 MessageBus 的 handlers 中集成
- `onChanged` 和 `onDeleted` 时保存历史快照和触发自动保存
- `onTextInput` 时仅触发自动保存（文字编辑会话内多次输入只计一次）
- 加载文件时清空历史记录并保存初始快照
