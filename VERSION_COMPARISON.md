# 版本对比：独立版 vs VS Code 插件版

## 概览

HTML 可视化编辑器提供两个版本，满足不同使用场景：

| 特性 | 独立版 | VS Code 插件版 |
|------|--------|----------------|
| 文件形式 | 单个 HTML 文件 | VS Code 扩展 |
| 依赖 | 零依赖 | 需要 VS Code |
| 安装 | 无需安装 | 需要安装插件 |
| 使用方式 | 浏览器打开 | VS Code 内打开 |
| 文件操作 | 手动打开/导出 | 自动保存 |
| 适用场景 | 快速编辑、演示 | 日常开发 |

## 独立版（html-editor.html）

### ✅ 优势

1. **零依赖**
   - 单个 HTML 文件，无需任何外部依赖
   - 可以通过 `file://` 协议直接打开
   - 适合离线使用

2. **便携性**
   - 可以通过邮件、U盘等方式分享
   - 在任何支持现代浏览器的设备上运行
   - 无需安装任何软件

3. **快速启动**
   - 双击即可打开
   - 无需等待编辑器启动
   - 适合快速编辑和演示

4. **独立性**
   - 不依赖特定编辑器
   - 可以作为独立工具使用
   - 适合教学和演示

### ❌ 限制

1. **文件操作**
   - 需要手动打开文件
   - 需要手动导出保存
   - 无法直接修改原文件

2. **工作流程**
   - 不集成到开发环境
   - 需要在浏览器和编辑器之间切换
   - 无法利用 VS Code 的其他功能

3. **功能限制**
   - 无法检测外部文件修改
   - 无法与版本控制集成
   - 无法使用 VS Code 的快捷键

### 📋 使用场景

- **快速编辑**：临时修改 HTML 文件
- **演示教学**：展示可视化编辑概念
- **离线使用**：无网络环境下编辑
- **分享工具**：发送给不使用 VS Code 的用户
- **备用方案**：VS Code 不可用时的替代方案

### 🚀 使用方法

```bash
# 方式 1：直接打开
双击 html-editor.html

# 方式 2：浏览器打开
在浏览器中打开 file:///path/to/html-editor.html

# 方式 3：本地服务器
python -m http.server 8000
# 访问 http://localhost:8000/html-editor.html
```

## VS Code 插件版

### ✅ 优势

1. **深度集成**
   - 完全集成到 VS Code 工作流
   - 使用 VS Code 的文件管理
   - 支持 VS Code 的快捷键和命令

2. **自动保存**
   - 编辑内容自动保存到文件
   - 无需手动导出
   - 支持 VS Code 的自动保存设置

3. **文件监控**
   - 检测外部文件修改
   - 提示重新加载
   - 避免内容冲突

4. **开发体验**
   - 右键菜单快速打开
   - 与其他扩展协同工作
   - 支持多文件编辑

5. **版本控制**
   - 与 Git 集成
   - 查看文件历史
   - 支持差异对比

### ❌ 限制

1. **依赖 VS Code**
   - 必须安装 VS Code
   - 需要安装插件
   - 无法在其他环境使用

2. **安装步骤**
   - 需要编译和打包
   - 需要安装依赖
   - 首次使用有学习成本

3. **资源占用**
   - 占用 VS Code 内存
   - 需要 webview 资源
   - 可能影响性能

### 📋 使用场景

- **日常开发**：在 VS Code 中编辑 HTML 文件
- **项目开发**：作为项目的一部分使用
- **团队协作**：团队统一使用 VS Code
- **持续编辑**：需要频繁编辑 HTML 文件
- **集成工作流**：需要与其他工具集成

### 🚀 使用方法

```bash
# 安装
cd vscode-extension
npm install
npm run compile

# 调试
按 F5 启动调试

# 打包安装
npm run package
code --install-extension html-visual-editor-0.1.0.vsix

# 使用
右键 .html 文件 → "用可视化编辑器打开"
```

## 选择建议

### 选择独立版，如果你：

- ✅ 需要快速编辑 HTML 文件
- ✅ 不想安装额外软件
- ✅ 需要在不同设备上使用
- ✅ 想要分享给他人使用
- ✅ 只是偶尔编辑 HTML
- ✅ 需要离线使用

### 选择插件版，如果你：

- ✅ 日常使用 VS Code 开发
- ✅ 需要频繁编辑 HTML 文件
- ✅ 希望自动保存修改
- ✅ 需要与版本控制集成
- ✅ 在团队中统一使用 VS Code
- ✅ 需要完整的开发工作流

## 两者可以共存

你可以同时保留两个版本：

1. **独立版**用于快速编辑和演示
2. **插件版**用于日常开发工作

它们共享相同的核心代码，功能完全一致，只是使用方式不同。

## 技术架构

两个版本共享相同的核心代码：

```
src/                    # 核心模块（共享）
├── color.js
├── inject.js
├── injected-script.js
├── property-panel.js
└── ...

html-editor.html        # 独立版（组装）
└── 内联所有核心模块

vscode-extension/       # 插件版（包装）
├── src/
│   ├── extension.ts    # VS Code 集成
│   └── htmlVisualEditorProvider.ts
└── 复用 html-editor.html
```

这种架构确保：
- 核心功能一致
- 独立开发和测试
- 易于维护和更新

## 总结

- **独立版**：轻量、便携、快速，适合临时使用
- **插件版**：集成、自动、强大，适合日常开发

根据你的使用场景选择合适的版本，或者两者都用！
